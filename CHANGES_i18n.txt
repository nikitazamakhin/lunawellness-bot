10. ТЕХНИЧЕСКИЕ ДЕТАЛИ
═══════════════════════════════════════════════════════════════

ФОРМАТ ПЕРЕВОДОВ:
- Все тексты в словаре TRANSLATIONS
- Структура: TRANSLATIONS[язык][ключ] = "текст"
- Поддержка форматирования через .format(**kwargs)
- Сохранены HTML теги (<b>, <i>)
- Сохранены эмодзи

ЯЗЫКОВЫЕ НАСТРОЙКИ:
- Язык запрашивается при первом /start
- Сохраняется в БД в поле users.language
- Используется во всех последующих сообщениях
- Применяется к воронке автоматически

FSM STATE:
- В booking процессе язык сохраняется в state.data
- Используется на всех этапах бронирования
- Не зависит от языка в БД (на случай изменений)

КЛАВИАТУРЫ:
- Все функции принимают параметр lang с default="ru"
- Тексты кнопок через get_text(lang, key)
- Callback_data остаются на английском (технические)

13. ПОДДЕРЖКА И ОБСЛУЖИВАНИЕ
═══════════════════════════════════════════════════════════════

ДОБАВЛЕНИЕ НОВЫХ ТЕКСТОВ:
1. Добавить ключ в TRANSLATIONS["ru"]
2. Добавить перевод в TRANSLATIONS["uz"]
3. Использовать get_text(lang, "new_key")

ИЗМЕНЕНИЕ СУЩЕСТВУЮЩИХ ПЕРЕВОДОВ:
1. Открыть bot/i18n.py
2. Найти нужный ключ
3. Изменить текст в нужном языке
4. Перезапустить бота

ДОБАВЛЕНИЕ НОВОГО ЯЗЫКА (например, EN):
1. Добавить TRANSLATIONS["en"] = {...}
2. Добавить кнопку выбора языка в language_selection()
3. Добавить обработку "lang:en" в language_selected()
