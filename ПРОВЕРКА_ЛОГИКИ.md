### 2. ВОРОНКА (12 ШАГОВ)

#### Шаги 1-7: ВИДЕО-КРУЖКИ (все пользователи)
```
Шаг 1:  Сразу     → Приветствие LUNA
Шаг 2:  +4 мин    → Пространство для себя
Шаг 3:  +10 мин   → Reformer Pilates
Шаг 4:  +15 мин   → Aerial Yoga
Шаг 5:  +30 мин   → Cycling
Шаг 6:  +60 мин   → SPA
Шаг 7:  +120 мин  → Детали атмосферы
```

**Логика:**
- ✅ Каждый шаг: проверка `is_booked()` → если да, СТОП
- ✅ Кружок отправляется первым (если есть file_id в БД)
- ✅ Пауза 3 сек после кружка
- ✅ Текст на выбранном языке
- ✅ Обновление funnel_step в БД
- ✅ Логирование события

#### Шаги 8-9: РАЗВЕТВЛЕНИЕ (has_phone vs no_phone)
```
Шаг 8:  +240 мин (8ч)    → РАЗВЕТВЛЕНИЕ
Шаг 9:  +960 мин (24ч)   → РАЗВЕТВЛЕНИЕ
```

**Логика разветвления:**
- ✅ Проверка `has_phone(telegram_id)` → смотрит в таблицу `bookings`
- ✅ Если has_phone → текст "спасибо, подготовка к занятию"
- ✅ Если no_phone → текст "попробуйте записаться" + кнопка
- ✅ Обе ветки получают сообщения

#### Шаги 10-12: ТОЛЬКО NO_PHONE
```
Шаг 10: +48ч (3 дня)   → ТОЛЬКО no_phone
Шаг 11: +48ч (5 дней)  → ТОЛЬКО no_phone
Шаг 12: +48ч (7 дней)  → ТОЛЬКО no_phone
```

**Логика:**
- ✅ Проверка `has_phone()` → если TRUE, SKIP
- ✅ Если FALSE → отправить текст + кнопка записи
- ✅ Шаги 10-12 НЕ отправляются пользователям с номером

### 3. БРОНИРОВАНИЕ

#### Процесс:
1. Пользователь нажимает "Записаться"
2. Выбирает направление (reformer/aerial/yoga/cycling/shape/spa)
3. Выбирает день недели
4. Вводит имя (текст)
5. Вводит телефон (кнопка "Поделиться" или текст)

#### Критические моменты:
- ✅ **МГНОВЕННОЕ подтверждение** при получении контакта
- ✅ Сообщение `contact_received` → БЕЗ reply keyboard
- ✅ Детали бронирования (направление, день, имя, телефон)
- ✅ Главное меню показывается после подтверждения
- ✅ Уведомление админу (всегда на русском)
- ✅ Установка `booked=1` в БД → **ОСТАНАВЛИВАЕТ ВОРОНКУ**
- ✅ Запись в таблицу `bookings` → делает `has_phone()` = TRUE

### 4. ОСТАНОВКА ВОРОНКИ

### 6. БАЗА ДАННЫХ

**Таблицы:**
```sql
users:
  - telegram_id (уникальный)
  - language (ru/uz)
  - funnel_step (0-12)
  - booked (0/1)
  - created_at

bookings:
  - telegram_id
  - direction, day, name, phone
  - status (new/confirmed/cancelled)
  - created_at

events:
  - telegram_id
  - event_type (start/language_selected/funnel_step_X/booking)
  - data
  - created_at

circles:
  - step (1-7)
  - file_id (Telegram file_id для переиспользования)
```

**Ключевые функции:**
- ✅ `is_booked(telegram_id)` → проверяет поле `booked`
- ✅ `has_phone(telegram_id)` → проверяет наличие записи в `bookings`
- ✅ `mark_booked(telegram_id)` → устанавливает `booked=1`
- ✅ `save_booking()` → сохраняет + вызывает `mark_booked()`

### 7. ВОЗОБНОВЛЕНИЕ ВОРОНКИ

- ✅ Функция `resume_funnels()` запускается в main.py
- ✅ Ищет пользователей: `funnel_step > 0 AND funnel_step < 10 AND booked = 0`
- ✅ Продолжает воронку с `funnel_step + 1`
